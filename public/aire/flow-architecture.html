AI-Native Architecture
Built on Cloudflare Workers Edge Computing + Multi-Dimensional AI Scoring
Edge
Cloudflare Workers
Serverless Edge Computing
  • Deploy ke 300+ cities worldwide (sub-100ms latency)
  • Zero cold start (V8 isolates, bukan containers)
  • Auto-scaling tanpa config (0 → 1M requests seamless)
Durable Objects
  • Stateful coordination untuk real-time features (live leaderboards)
  • Strong consistency across edge locations
  • WebSocket support untuk conversational AI assessments
KV Storage
  • Cache assessment templates globally (instant retrieval)
  • Eventual consistency (perfect untuk read-heavy workloads)
  • Unlimited reads, pay-per-write pricing model
Kenapa Workers over traditional backends: Assessment scoring butuh instant response (<200ms). Traditional servers di single region = 500ms+ latency untuk global users. Edge computing distribute logic globally, scoring happens di location terdekat candidate.
AI Engine
Multi-Dimensional BARS Scoring
Research-Grounded Rubrics
  • BARS (Behaviorally Anchored Rating Scales) untuk tiap competency
  • Rubrics derived dari I-O psychology meta-analyses
  • Per-dimension scoring (System Design: 8/10, Collaboration: 6/10)
Conversational AI Assessments
  • Natural language interaction (chat-based, bukan static forms)
  • Adaptive follow-ups based on candidate responses
  • Context-aware probing (like talking to experienced interviewer)
Video Analysis Engine
  • Integrity monitoring (tab switching, multiple faces detection)
  • Transcript generation + semantic scoring
  • Unified scorecard across CV + assessment + video
Kenapa multi-dimensional scoring: Single score ("Candidate A: 75/100") hide nuance. Multi-dimensional breakdown (System Design: 9, Leadership: 5, Collaboration: 7) reveal actual strengths/gaps. HR make better decisions dengan clearer signal.
Database
D1 (SQLite at Edge) + R2 Storage
D1 SQL Database
  • SQLite running di Workers edge (no separate DB server)
  • ACID guarantees untuk critical writes (candidate submissions)
  • Read replicas globally distributed untuk low-latency queries
R2 Object Storage
  • Video recordings, CV files, assessment artifacts
  • S3-compatible API, zero egress fees
  • Automatic CDN caching untuk frequently-accessed files
Analytics Engine
  • Workers Analytics untuk real-time usage tracking
  • Bias audit aggregations (score drift by demographics)
  • Funnel analysis (invited → started → completed)
Kenapa edge database: Traditional setup = frontend → API server → database (3 network hops, 200-500ms). D1 at edge = single hop dari user ke Workers yang punya database access locally (<50ms total latency).
Frontend
React + Vite SPA
Framework
  • React 18 dengan Vite bundler (HMR <100ms)
  • TanStack Query untuk server state management
  • React Router 6 untuk SPA navigation
Styling & UI
  • Tailwind CSS untuk rapid UI development
  • Radix UI primitives (accessible components)
  • Framer Motion untuk micro-interactions
Real-Time Features
  • WebSocket connections via Durable Objects
  • Optimistic updates untuk snappy UX
  • Server-Sent Events untuk live leaderboard updates
Kenapa SPA over SSR: Assessment platform adalah behind-login app, bukan marketing site. SEO not a concern. SPA give us instant navigation, optimistic updates, dan rich interactions tanpa full page reloads. Simpler architecture, better UX.
Key Architecture Decisions
Edge Computing over Traditional Backends
Traditional setup (EC2, GCP Compute) = single-region deployment, 500ms+ latency untuk global users, manual scaling config, cold starts. Workers = global deployment otomatis, <100ms latency worldwide, zero cold starts, auto-scaling tanpa config. Trade-off: vendor lock-in, tapi performance gain massive.
Serverless over Containers
Containers (Kubernetes, ECS) = operational complexity, manual scaling policies, resource provisioning. Serverless = write code + deploy, platform handle scaling/availability. Trade-off: cold starts (mitigated dengan Workers' V8 isolates), vendor limitations. For assessment platform dengan spiky traffic, serverless win.
Multi-Dimensional Scoring over Single Score
Single composite score hide nuance ("Candidate A: 82/100" — strong at what?). Multi-dimensional breakdown (System Design: 9/10, Communication: 6/10, Leadership: 7/10) reveal actual profile. Trade-off: more complex untuk candidate, tapi way more actionable untuk hiring decisions.
Conversational AI over Static Forms
Static forms = same questions untuk semua, no context, rigid. Conversational AI = adaptive follow-ups, natural interaction, context-aware probing. Trade-off: higher cost per assessment ($8 vs nearly-free static forms), tapi candidate experience jauh lebih engaging dan signal quality higher.
Research-Grounded Rubrics over Keyword Matching
Keyword matching ("response contains 'microservices' → +5 points") = gameable, surface-level. BARS rubrics grounded in I-O psychology = nuanced scoring ("demonstrates understanding of CAP theorem tradeoffs" → Level 4/5). Trade-off: rubric development upfront effort, tapi scoring validity way higher.
Pay-Per-Use over SaaS Subscriptions
Annual SaaS ($1,700/year TestGorilla) = upfront commitment, unused licenses, credit expiry. Pay-per-use ($8/completed assessment) = zero lock-in, pay only when hiring. Trade-off: less predictable revenue untuk us, tapi customer acquisition friction way lower (free to start, no credit card).
AI-Native vs AI-Bolted Architecture
AI-Bolted (Legacy Competitors)
Pattern: Legacy CRUD app (Rails/Django) + AI feature bolted on top (OpenAI API call untuk CV screening).
Result: AI feels tacked-on, keyword matching dengan AI label, no conversational flow, scoring inconsistent.
Architecture: Monolithic backend, single-region, traditional SQL, batch processing.
AI-Native (Aire)
Pattern: Built from scratch dengan AI as core. Conversational assessments, multi-dimensional scoring, adaptive follow-ups default.
Result: Natural interaction, nuanced scoring, research-grounded rubrics bukan keyword matching.
Architecture: Edge computing (Workers), distributed globally, real-time scoring, WebSocket-native.
