Key Architecture Decisions & Rationale
Monorepo vs. Multi-Repo
Monorepo
Problem: Web + mobile + backend di separate repos = dependency hell, slow iteration, susah refactor shared logic.
Solution:

Monorepo dengan pnpm workspaces. Shared packages (@preppy/ui, @preppy/api-client) symlinked. Satu PR change web + mobile + backend sekaligus. CI run all tests together.

Server-Side Rendering (SSR)
CSR Only
Problem: SSR improve SEO dan perceived load time, tapi add complexity (Next.js, hydration bugs, higher server costs).
Solution:

Pure CSR (Vite SPA). SEO handled via prerendering (Prerender.io) untuk marketing pages. App di balik login anyway. Faster iteration > marginal SEO gains.

State Management
TanStack Query
Problem: Redux = boilerplate hell. Zustand = manual cache invalidation. Context API = prop drilling.
Solution:

TanStack Query untuk server state (auto-caching, background refetch, optimistic updates). Zustand untuk minimal client state (theme, auth token). Best of both worlds.

Real-Time Updates
Polling + Socket.io
Problem: Leaderboards butuh real-time updates. WebSockets = connection management headaches at scale.
Solution:

Hybrid: Polling (tiap 30s) untuk most data. Socket.io cuma untuk leaderboard page (high engagement). Avoid keeping 10k idle connections open 24/7.

Authentication Strategy
JWT + Refresh
Problem: Sessions = sticky backend (no horizontal scaling). Pure JWT = can't revoke tokens (security risk).
Solution:

Short-lived JWT (15 min) + refresh token di httpOnly cookie. Redis stores refresh tokens (bisa revoke). Stateless + secure + scalable.

Offline-First Architecture
Partial
Problem: Full offline-first (CRDTs, conflict resolution) = massive complexity. Tapi user belajar on shaky mobile connections.
Solution:

Cache read-only content (lessons, questions) di IndexedDB. Writes queue di local storage, retry on reconnect. Good enough untuk 95% use cases.

Architecture Tradeoffs & Learnings
React vs. React Native
Pilih: React + Capacitor
Tradeoff: Slightly worse native performance, tapi 10x faster iteration. Untuk use case kita (forms, text, animations), perf difference negligible. Bisa always rewrite bottlenecks di native later.
Tailwind vs. CSS-in-JS
Pilih: Tailwind
Tradeoff: HTML jadi verbose, tapi zero runtime cost. Styled-components add 16KB bundle + runtime parsing. Untuk mobile, tiap KB matters. Tailwind wins.
Microservices vs. Monolith
Pilih: Modular monolith
Tradeoff: Less "cool", tapi way simpler. AI service separate (Python), rest satu Node app. No inter-service networking hell. Bisa extract microservices later kalau butuh.
GraphQL vs. REST
Pilih: REST
Tradeoff: Over-fetching data, tapi way simpler untuk cache (Cloudflare CDN). GraphQL = N+1 queries, complex caching, overkill untuk simple CRUD kita. REST + TanStack Query = fast enough.
TypeScript vs. JavaScript
Pilih: TypeScript everywhere
Tradeoff: Slower initial setup, tapi catch 30% bugs at compile time. Refactoring jadi fearless. Autocomplete save hours. Non-negotiable untuk team >1 orang.
Self-Host vs. Cloud
Pilih: Cloud (Vercel + Railway)
Tradeoff: Vendor lock-in risk, tapi zero DevOps time. Self-hosting = managing Kubernetes, scaling, security patches. Sebagai 2-person team, cloud adalah 10x ROI.
