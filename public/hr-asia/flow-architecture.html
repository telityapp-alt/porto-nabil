Platform Architecture
Static-first publishing dengan dynamic personalization layer
Design Philosophy
Content-First, Performance-Obsessed
HR Asia adalah content platform, bukan app platform. Architecture priorities: fast page load (under 2s), zero JS untuk core content, SEO-optimized structure, dan easy content authoring workflow. Dynamic features (tools, personalization) layer on top tanpa slow down content delivery.
Core Principles
  • Static by default: News articles, regulation vault, templates = pre-rendered HTML (Next.js SSG atau Astro)
  • Progressive enhancement: Interactive tools (graders, calculators) load JS only when needed
  • Edge-first delivery: Static assets served dari CDN edge locations (sub-100ms TTFB di Asia)
  • Markdown-driven content: Writers author dalam markdown, Git-based version control, auto-deploy on merge
Frontend Stack
Static Site Generation + React Islands
Kami pake Next.js (App Router) dengan static export untuk content pages, dan React Server Components untuk interactive sections. Majority of pages ship zero client-side JS. Tools (Job Description Grader, Salary Check, etc.) hydrate as React islands — JS bundle lazy loaded only when user interact.
Technology Choices
Framework
Next.js 14 (App Router) — Static export mode untuk content, RSC untuk tools
Styling
Tailwind CSS — Aire brand colors sebagai custom theme (Ink, Cream, Olive, Mustard, dll)
Typography
Lora (display headings) + Inter (body copy) — self-hosted via next/font untuk performance
Content
MDX untuk articles — frontmatter metadata, reusable components (callouts, code blocks, embedded tools)
State
Zustand untuk client-side state (newsletter subscription, tool results caching)
Backend & Data
Serverless + Edge Functions
Backend operations minimal: newsletter signups, tool execution (grading logic, salary benchmarking), analytics tracking. Semua run as serverless functions di edge untuk low-latency response. Database hanya untuk newsletter subscribers dan tool usage logs — majority of content fully static.
Backend Architecture
Functions
Cloudflare Workers (atau Vercel Edge Functions) — sub-50ms execution di Asia POPs
Database
Cloudflare D1 (SQLite at edge) — newsletter subscribers, tool usage analytics
CMS
Git-based (GitHub repo) — content authors commit markdown, CI/CD auto-deploy to production
Email
Resend API (atau Cloudflare Email Workers) — newsletter delivery, transactional emails
Content Pipeline
Markdown → HTML Publishing Flow
Content workflow optimized untuk speed dan consistency. Writers draft di markdown (VS Code atau any editor), commit ke Git branch, open PR untuk review. On merge to main: CI/CD trigger static build, generate HTML pages, deploy ke CDN. Zero manual deployment steps. Content goes live dalam under 5 minutes dari merge.
Publishing Steps
  • Step 1 — Author: Write article/regulation page dalam markdown dengan frontmatter (title, date, country, tags)
  • Step 2 — Review: Open GitHub PR, automated checks run (spelling, broken links, metadata validation)
  • Step 3 — Build: On merge, Next.js static export generate HTML + CSS + minimal JS bundles
  • Step 4 — Deploy: Cloudflare Pages (atau Vercel) deploy ke edge, invalidate CDN cache, content live globally
  • Step 5 — Notify: Slack notification ke team + analytics ping (new content published)
SEO & Discoverability
Structured Data + Fast Core Web Vitals
SEO critical untuk discovery: majority of traffic datang dari organic search ("labor law Indonesia", "HR templates Malaysia", etc.). Kami optimize untuk Core Web Vitals (LCP under 2s, CLS near 0, FID under 100ms), semantic HTML, structured data markup (Article, FAQPage schema), dan internal linking strategy.
SEO Implementation
  • Metadata: Dynamic Open Graph tags, Twitter Cards, canonical URLs per page
  • Schema.org: Article markup untuk news, FAQPage untuk regulation vault, HowTo untuk templates
  • Sitemap: Auto-generated XML sitemap dengan lastmod timestamps, priority hints
  • Robots.txt: Allow all crawlers, explicit sitemap reference
  • Performance: Image optimization (next/image), font subsetting, critical CSS inline
Analytics & Tracking
Privacy-First Metrics
Kami track user behavior untuk understand content resonance dan optimize funnel, tapi respect privacy. No third-party trackers (Google Analytics replaced dengan self-hosted Plausible atau Cloudflare Web Analytics). Event tracking: page views, tool usage, newsletter signups, Aire CTR. Data stored 90 days max.
Key Metrics Tracked
  • Content: Page views per article/regulation page, time on page, scroll depth
  • Tools: Tool execution count, completion rate, result satisfaction (thumbs up/down)
  • Funnel: Newsletter signup conversion per entry point, Aire CTR dari newsletter footer
  • Performance: Core Web Vitals real user monitoring (RUM), TTFB per region
Architecture Trade-offs
Why static-first: Content changes infrequently (weekly newsletter, occasional regulation updates). Pre-rendering everything = instant page loads, zero database queries on page view, bulletproof uptime. Trade-off: real-time personalization limited (can't show "you read 3 articles this week" without client-side JS). We accept this — content delivery speed > personalization features untuk discovery phase. Why edge functions: Tools need server-side logic (grading algorithms, salary data lookups). Edge execution = consistent sub-100ms response globally, no cold starts. Trade-off: limited compute time (Cloudflare Workers max 50ms CPU). We design tools untuk finish within budget — heavy ML inference offloaded ke Aire backend if needed.
